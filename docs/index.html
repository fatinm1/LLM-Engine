ZERO COPY

Memory-mapped
model loading

The GGUF binary is opened with mmap — tensor weight pointers point directly into the mapped region. Loading an 808 MB model never copies a single byte into heap memory. The OS pages in only the weights that are actually touched during inference.

ARCHITECTURE

Transformer forward pass,
from scratch.

Each token generation step runs through a full Llama transformer — embedding lookup, 16 layers of GQA attention and SwiGLU FFN, RMSNorm, and output projection. No framework, no abstraction layer.

See the source code

KV Cache

O(n) generation

Past K and V tensors are stored per layer. Each new token attends to cached keys instead of recomputing the full sequence.

[t=0] K₀ V₀ ← cached
[t=1] K₁ V₁ ← cached
[t=2] K₂ V₂ ← cached
[t=n] Kₙ Vₙ ← attend here

Grouped Query Attention

32 heads / 8 KV heads

Each KV head is shared by 4 query heads — reducing cache size 4× vs standard multi-head attention. 

// GQA head mapping
kv_h = query_head / 4;
// 32 Q heads → 8 KV heads
// 4× smaller KV cache

SwiGLU FFN

gate = w_gate @ x_norm
up   = w_up   @ x_norm
out  = w_down @ (silu(gate) * up)
// ff_dim = 8,192

QUANTIZATION

Mixed precision,
handled automatically.

Q3_K_M

3-bit weights with K-means quantization. Used for attention weights (wq, wk, wv, wo) in Llama 3.2. 110 bytes per 256-element block.

Block layout (110 bytes)

hmask·32 qs·64 scales·12 d·2

← 110 bytes total →

Q4_K_S

4-bit symmetric quantization. Used for FFN weights (ffn_gate, ffn_up, ffn_down) in Llama 3.2. 144 bytes per 256-element block.

Block layout (144 bytes)

scales·12 d·2 dmin·2 qs·128

← 144 bytes total →

Full precision norms

RMSNorm weights and embedding lookups stay in F32 for numerical stability. Q8_0 provides an 8-bit fallback at 34 bytes per 32-element block.

output_norm.weight F32
token_embd.weight Q4_K_S
attn_norm.weight F32
